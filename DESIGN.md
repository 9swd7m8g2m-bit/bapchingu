# 🍱 밥친구 조편성 자동화 — 기획안

## 1. 한 줄 요약
전 직원 명단에서 **4~5명씩 밥친구 조를 자동 편성**하고, 결과를 표 이미지로 내보내는 웹앱.
연차·출장·팀이동·중도합류를 **누구나 클릭 몇 번으로 반영**하고 다시 이미지를 뽑을 수 있다.

## 2. 해결하려는 문제
- 코드를 직접 돌려야 함 → 브라우저 버튼 클릭으로
- 한 명만 할 줄 앎(휴가 가면 마비) → URL만 알면 **아무나** 조작
- 연차/출장/팀이동/중도합류 → 코드 수정·재실행이 필요 → 화면에서 토글·이동 후 재생성
- 명단이 코드에 하드코딩 → 화면에서 명단 관리

## 3. 기능 (MVP)
1. **명단 관리** — 이름·부서·재직여부 추가/수정/삭제, 팀장 풀 지정
2. **이번 주 상태 토글** — `참여 / 연차·출장(제외) / 샐러드팀`
3. **자동 편성** — "조 생성" 버튼
   - 조당 인원 **고정 4~5명**, 조 개수는 참여 인원에 따라 자동 계산
   - 매주 랜덤 셔플
   - **같은 두 사람이 3주 연속 같은 조가 되지 않게**만 회피 (가끔 겹치는 건 허용)
   - 팀장은 **고정 팀장 풀**에서 각 조에 1명씩 배치
4. **수동 보정** — 다른 조로 이동, 중도합류 추가
   - 신규/이동 표시 (색·아이콘은 디자인 톤에 맞춰 결정)
5. **이미지 내보내기** — 표를 PNG로 다운로드 (html2canvas) → 카톡/팀즈에 붙여넣기
6. **하단 메모 줄** — "OOO: 오늘 첫 출근입니다" 등 자유 메모

## 디자인 톤
모던 카드 — 연한 파스텔 배경, 둥근 모서리 카드, 부드러운 그림자.
조 제목에 🍱 이모지, 팀장은 👑 아이콘으로 표시. 조별 파스텔 색 구분.

## 4. 화면 구성
```
[명단 관리] [이번 주 상태] [조 생성] [이미지 저장]
──────────────────────────────────────────
 ┌Team1┐ ┌Team2┐ ┌Team3┐ ...   ← 스크린샷과 같은 카드 그리드
 │[팀장]│ │[팀장]│ │[팀장]│
 │멤버  │ │멤버  │ │멤버  │
 └──────┘ └──────┘ └──────┘
 [하단 메모 입력칸]
```
이 그리드 영역을 캡처해 이미지로 저장.

## 5. 편성 알고리즘
1. `참여` 상태인 사람만 풀에 포함 (연차·출장·샐러드팀 제외)
2. 참여 인원 ÷ 4~5 로 **조 개수 자동 산정**, 4명/5명 조를 섞어 배분
3. 팀장 풀에서 각 조에 1명씩 배치
4. 나머지를 셔플하되, **직전 2주 모두 같은 조였던 페어**는 같은 조에 다시 넣지 않음
   (= 3주 연속 금지). 그 외 겹침은 허용
5. 조건을 만족하는 조합이 안 나오면 셔플 재시도(최대 N회) 후 가장 위반 적은 결과 채택

## 6. 데이터 저장 — Supabase (공유)
누가 접속해도 같은 명단·주간기록을 보도록 Supabase 사용.

테이블 스케치:
- `members` — id, name, dept, is_active, is_leader
- `weeks` — id, week_start_date, memo
- `assignments` — id, week_id, team_no, member_id, status('member'|'leader'|'new'|'moved')
- 페어링 회피용: `assignments`에서 최근 weeks 조회해 같은 team_no 동석 이력 계산

## 7. 기술 스택
- 단일 `index.html` (바닐라 JS) — 기존 프로젝트와 동일
- 이미지 저장: html2canvas (CDN)
- 데이터: Supabase (@supabase/supabase-js CDN)

## 8. 개발 순서
1. 명단 하드코딩 → 카드 그리드 렌더 + 이미지 저장 (결과부터 눈으로 확인)
2. 편성 알고리즘 (셔플 + 팀장 고정)
3. 상태 토글·수동 이동·신규/이동 색상 표시
4. Supabase 연동 + 최근 회피 로직
5. 명단 관리 UI
